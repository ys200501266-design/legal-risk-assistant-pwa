# User Flow：从输入到结果

## 1. 打开 PWA

用户通过手机浏览器打开公开链接。首页展示产品名称、用途、三种输入入口和常见场景。

## 2. 选择输入方式

用户选择拍照识别、语音输入或文字咨询。

- 拍照识别：上传合同、票据、聊天记录截图等图片材料
- 语音输入：点击语音入口，后续可接入 Web Speech API 或录音上传
- 文字咨询：直接输入问题描述

## 3. 选择场景

用户选择租房、劳动、消费、合同或其他场景。场景会影响后续检索范围和风险提示模板。

## 4. 开始分析

点击“开始分析”后，系统进行以下模拟流程：

1. OCR 或 ASR 转写输入内容
2. 提取关键事实
3. 根据场景和事实检索相关法律依据
4. 生成风险等级和解释
5. 输出下一步建议

如果用户没有输入文字或上传图片，页面会提示先补充问题描述或材料。

## 5. 查看结果

结果页展示：

- 识别到的问题
- 事实摘要
- 风险等级
- 相关法律依据卡片
- AI 解释
- 下一步建议
- 免责声明

## 6. 行动或返回

用户可以根据建议保存证据、补充材料、协商、投诉或咨询律师，也可以返回首页重新输入新的问题。

## 7. 添加到主屏幕

当浏览器支持 PWA 安装时，首页底部展示“添加到手机桌面”。如果浏览器不支持自动安装提示，则展示“可通过浏览器菜单添加到主屏幕”。
