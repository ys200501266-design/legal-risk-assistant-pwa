# PRD：法律风险识别助手 PWA Demo

## 项目背景

普通用户在租房、劳动、消费、合同等生活场景中经常遇到法律相关问题，但很难快速判断应补充哪些事实、可能涉及哪些法律依据、下一步该如何行动。传统搜索需要用户自己筛选信息，生成式 AI 又可能出现武断判断或编造法条的风险。

本项目以 PWA 形式展示一个手机端优先的法律风险识别助手 Demo：用户通过拍照、语音或文字描述问题，系统先提取事实，再匹配法律依据，最后输出风险等级、相关法条和行动建议。

## 目标用户

- 遇到生活法律问题但尚未准备咨询律师的普通用户
- 希望快速整理事实、证据和下一步行动的用户
- 需要在手机端低门槛使用法律信息检索工具的用户
- 作品集评审者、AI 产品经理面试官、前端工程师面试官

## 用户痛点

- 不知道自己的问题属于什么法律场景
- 搜索结果碎片化，难以知道哪些法条真正相关
- AI 回答可能直接判断“违法”，但没有依据或证据链
- 用户不知道下一步应该保存哪些证据、如何协商或投诉
- 普通网页不像 App，手机端使用和留存体验较弱

## 核心功能

- 拍照识别入口：上传合同、票据、聊天记录等图片材料
- 语音输入入口：面向移动端的口述问题入口
- 文字咨询入口：用户直接输入生活法律问题
- 场景选择：租房、劳动、消费、合同、其他
- 事实提取：把用户描述拆解为可核验事实
- 法律依据引用：展示法律名称、条文编号、摘要和来源链接
- 风险分级：使用低风险、中风险、高风险、无法判断，而非绝对违法判断
- 下一步建议：保存证据、补充材料、协商、投诉、咨询律师
- PWA 安装：支持添加到手机主屏幕

## MVP 范围

- 使用 React + TypeScript + Vite 完成前端 Demo
- 使用 mock legal_cases.json 模拟 RAG 检索结果
- 使用 vite-plugin-pwa 配置 manifest、service worker 和基础离线缓存
- 实现 Home、Analyze、Result 三个核心页面
- 完成风控文档、Prompt 设计和 README

## 非目标

- 不接入真实 OCR、ASR、LLM 或法律数据库
- 不提供正式法律意见
- 不进行案件胜率预测
- 不替代律师、司法机关、劳动仲裁或消费者保护机构判断
- 不处理用户登录、付费、案件管理等复杂业务

## 风险控制

- 输出风险等级，不输出“已确认违法”等绝对结论
- 结果必须展示相关法律依据
- 未找到法条依据时显示“未找到明确法律依据，建议补充信息或咨询专业人士”
- 强制展示免责声明
- 对地区差异、信息不足、证据不足进行提示
- 法条来源链接指向可信法律法规公开平台

## 成功指标

- 手机端首屏 5 秒内理解产品用途
- 用户能在 3 步内完成输入并看到分析结果
- 结果页包含事实摘要、风险等级、法条依据、AI 解释、下一步建议
- PWA manifest 与 service worker 正常生成
- 可部署到 Vercel 并通过手机浏览器访问
- 作品集读者能看出 RAG、OCR、语音、幻觉控制和法律引用设计能力
