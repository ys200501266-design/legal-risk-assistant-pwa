# 法律风险识别助手

面向普通用户的 AI 法律风险初筛 PWA Demo。用户可以上传图片或输入文字，系统识别可能涉及的法律风险，并输出风险等级、判断原因、法律依据和下一步建议。

这个项目不替代律师，也不直接给出“合法 / 违法”的绝对结论。它更像一个生活法律问题的第一步判断工具，帮助用户先看清风险、依据和下一步动作。

## 核心功能

- 图片 / 文字输入。
- 场景选择：合同、租房、消费、兼职、聊天记录、其他。
- 识别中状态：内容识别、依据匹配、风险结论生成。
- 结果页：风险等级、可能涉及的问题、判断原因、法律依据、建议动作。
- 继续追问：提供常见追问入口和谨慎的模拟回复。
- 保存结果：点击后显示“已保存”，并写入本地历史记录。
- 历史记录：使用 localStorage 保存模拟数据，刷新后仍可查看。

## 页面说明

1. 首页：展示产品名称、说明、上传图片和输入文字两个入口。
2. 输入页：包含图片上传区域、文本输入框、场景选择标签和开始识别按钮。
3. 识别中页：用步骤状态表现内容识别、法律依据匹配和风险结论生成。
4. 结果页：突出中风险、租房押金退还争议、判断原因、法律依据和第一条建议动作。
5. 追问页：展示可继续追问的问题、输入框和一条谨慎的模拟回复。
6. 历史记录页：展示已保存记录；没有保存时显示空状态。

## 本地运行方式

```bash
npm install
npm run dev
```

如果当前电脑没有把 Node / npm 加入 PATH，也可以使用 pnpm 或项目环境中的 Node 运行。项目本身不依赖后端 API。

## 构建方式

```bash
npm run build
```

构建产物在：

```text
dist/
```

## 线上部署方式

方案 A：独立部署 Demo

1. 将本项目推送到 GitHub。
2. 在 Netlify 或 Vercel 新建站点。
3. Build Command 填 `npm run build`。
4. Publish Directory 填 `dist`。
5. 部署成功后，把 Demo 链接放到作品集项目页。

方案 C：挂到作品集主站

1. 在本项目执行 `npm run build`。
2. 将 `dist/` 内文件复制到作品集主站的 `public/demos/legal-risk-assistant/`。
3. 在作品集项目卡片中将 Demo 链接设置为 `/demos/legal-risk-assistant/`。
4. 重新部署作品集主站。

当前更适合投递的方案是方案 C：HR 先进入作品集，再从 Projects 页面看到项目说明和可点击 Demo。
